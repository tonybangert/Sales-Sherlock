# Customizing prompts

Every section of a Sherlock brief is generated by a single prompt template. All seven live in `src/sherlock/prompts/`:

- `01_tldr.md`
- `02_company_snapshot.md`
- `03_person_role.md`
- `04_three_things.md`
- `05_opening_questions.md`
- `06_discovery_questions.md`
- `07_risk_flag.md`

Each template uses exactly one placeholder: `{{research}}`. At brief-generation time, the full dossier (LinkedIn paste, company website summary, Apollo enrichment if available) gets substituted in.

## Override prompts without forking

The simplest way to customize is to copy the bundled prompts to a directory you own, edit them there, and tell Sherlock to use that directory:

```bash
# 1. Find the bundled prompts directory
PROMPTS_DIR=$(python -c "import sherlock.prompts, os; print(os.path.dirname(sherlock.prompts.__file__))")

# 2. Copy them somewhere you control
mkdir -p ~/.sherlock/prompts
cp "$PROMPTS_DIR"/*.md ~/.sherlock/prompts/

# 3. Edit any file
$EDITOR ~/.sherlock/prompts/06_discovery_questions.md

# 4. Tell Sherlock to load yours instead of the defaults
export SHERLOCK_PROMPTS_DIR=~/.sherlock/prompts
sherlock brief --interactive
```

Sherlock checks the override directory first for each section. If a file isn't there, it falls back to the bundled version — so you only have to override the sections you actually want to change.

## Common customizations

### Industry-specific discovery

Out of the box, the discovery questions probe for revenue-intelligence, AI enablement, and operational efficiency signals. If you sell into security, fintech, healthcare, or any other vertical, rewrite `06_discovery_questions.md` to surface signals your buyer actually cares about.

### Executive vs. practitioner opening

The opening questions in `05_opening_questions.md` are calibrated for a thoughtful working-level conversation. If you primarily call into VP+ executives, edit the file to push the questions toward strategic-level framings — board priorities, fiscal-year planning, M&A speculation.

### Inject your own context

The most powerful customization is adding a short framing block at the top of every prompt that describes who *you* sell, what value prop you lead with, and what your typical deal looks like. Then every brief section is generated with that context in mind. Example header to add to all seven prompts:

```
You are preparing material for a seller from PerformanceLabs.AI, which builds
AI tools for revenue teams. Their typical engagement is a 4-6 week deployment
of a custom AI workflow inside the customer's RevOps stack, priced $25-75K.
They sell to VP RevOps, CROs, and Heads of Sales Operations.

Below is the research dossier:

{{research}}

[... rest of original prompt ...]
```

### Different brief lengths

Each section's prompt asks for a specific length (e.g. "2-3 sentences" or "4-6 sentences"). If you want shorter or longer briefs, change those instructions inline.

## Tips

- **Test against real prospects.** After any prompt change, generate three briefs against real (or anonymized) profiles. If two of three are noticeably better than the bundled prompt, ship it.
- **Keep `{{research}}` in.** Every prompt must contain the placeholder exactly once, or that section will fail silently.
- **Don't over-instruct.** Long, prescriptive prompts often produce worse output than short, sharp ones. Resist the urge to explain everything.
- **Version-control your overrides.** Once you've tuned the prompts, commit them somewhere your team can share. They become the most valuable artifact in the whole setup.
